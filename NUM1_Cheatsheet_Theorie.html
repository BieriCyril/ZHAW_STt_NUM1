1 · Kondition eines linearen Gleichungssystems
LGS / Fehler
Worum geht es?

Wie stark verstärkt das LGS einen Fehler in den Eingabedaten? Eine schlecht konditionierte Matrix vergrössert kleine Fehler in b zu grossen Fehlern in x.

Schlagwörter in der Aufgabe
  • relativer Fehler
  • 1-Norm / ∞-Norm
  • Näherungslösung x̃ ≈ x
  • rechte Seite b̃ ≈ b
  • A⁻¹ wird gegeben
Gegeben / Gesucht
Gegeben Matrix A, A⁻¹, rel. Fehler von x̃

Gesucht Konditionszahl cond(A)
Gesucht minimaler rel. Fehler von b̃
Kochrezept
  1. Norm von A berechnen
    1-Norm = grösste Spaltensumme der Beträge
    ∞-Norm = grösste Zeilensumme der Beträge
  2. Norm von A⁻¹ berechnen (gleiche Norm!)
  3. Konditionszahl:
    cond(A) = ‖A‖ · ‖A⁻¹‖
  4. Fehlerbeziehung:
    ‖Δx‖/‖x‖ ≤ cond(A) · ‖Δb‖/‖b‖Umformen nach ‖Δb‖/‖b‖:
    rel. Fehler(b) ≥ rel. Fehler(x) / cond(A)
1-Norm Beispiel: Spalte 1 hat Summe 11, Spalte 2 hat Summe 6 → ‖A‖₁ = 11
Grosse Konditionszahl (z.B. 10⁶) = schlecht konditioniert = kleine Fehler in b führen zu grossen Fehlern in x
Immer dieselbe Norm für A und A⁻¹ verwenden!
2 · Lineares Randwertproblem (Finite-Differenzen-Methode)
RWP / FDM
Worum geht es?

Eine DGL mit Bedingungen an beiden Enden des Intervalls (nicht Anfangsbedingungen!). Man diskretisiert das Intervall und ersetzt Ableitungen durch Differenzenquotienten → LGS.

Schlagwörter in der Aufgabe
  • Randwertproblem (RWP)
  • y(a) = ..., y(b) = ... (Randbed.)
  • zentrale Differenzenquotienten
  • Ordnung 2
  • äquidistante Knoten x₀…x_{N+1}
  • Schablone / Diskretisierung
  • Finite-Differenzen-Methode
Gegeben / Gesucht
Gegeben DGL y'' + p(x)y' + q(x)y = r(x)
Gegeben Randwerte y(a), y(b)
Gegeben Anzahl Schritte N

Gesucht Schablone (Koeffizienten)
Gesucht Systemmatrix und rechte Seite
Gesucht maximaler Fehler vs. exakte Lösung
Kochrezept
  1. Schrittweite: h = (b − a) / (N+1), Knoten: xᵢ = a + i·h für i = 0…N+1
  2. Ableitungen ersetzen (zentral, Ordnung 2): y'(xᵢ) ≈ (yᵢ₊₁ − yᵢ₋₁) / (2h) y''(xᵢ) ≈ (yᵢ₋₁ − 2yᵢ + yᵢ₊₁) / h²
  3. In DGL einsetzen → pro innerem Knoten eine Gleichung mit (yᵢ₋₁, yᵢ, yᵢ₊₁)
  4. Schablone aufschreiben: Koeffizienten vor yᵢ₋₁, yᵢ, yᵢ₊₁ (und rechte Seite)
  5. LGS aufstellen: tridiagonale Matrix, Randwerte in rechte Seite schieben (yᵢ₋₁ und yᵢ₊₁ am Rand sind bekannt!)
  6. LGS lösen → innere Werte y₁…y_N
  7. Fehler: max|yᵢ_num − yᵢ_exakt| über alle Knoten
Konvergenzordnung prüfen: Fehler sollte sich bei Halbierung von h ungefähr vierteln (Ordnung 2)
3 · Lineare Ausgleichsrechnung (Kleinste Quadrate / QR)
Least Squares
Worum geht es?

Mehr Messdaten als Unbekannte → überbestimmtes System, keine exakte Lösung. Gesucht sind Parameter, die die Summe der quadratischen Abweichungen minimieren.

Schlagwörter in der Aufgabe
  • Methode der kleinsten Quadrate
  • Datenpunkte (xᵢ, yᵢ)
  • Modell y = a·f₁(x) + b·f₂(x) + ...
  • Parameter a, b, c bestimmen
  • QR-Zerlegung verwenden
  • Summe der Fehlerquadrate
  • lineares Modell in den Parametern
Gegeben / Gesucht
Gegeben Datenpunkte (CSV oder Tabelle)
Gegeben Modellstruktur (Basisfunktionen)

Gesucht Parameter (a, b, c, ...)
Gesucht Summe der Fehlerquadrate ‖Ax − b‖²
Kochrezept — QR-Zerlegung
  1. Designmatrix A aufstellen (Grösse: Anzahl Datenpunkte × Anzahl Parameter)
    Jede Spalte = eine Basisfunktion ausgewertet an allen xᵢ
    A[i,j] = fⱼ(xᵢ)
  2. Rechte Seite b = Vektor der yᵢ-Werte
  3. QR-Zerlegung von A: A = Q·R
  4. Lösung durch Rückwärtseinsetzen: R · p = Qᵀ · b → p = R⁻¹(Qᵀb)
  5. Fehlerquadrate: residuen = b − A·p, SSE = Σ residuenᵢ²
Sonderfall: Linearisierung (z.B. Aufgabe 4)
  1. Modell nicht linear in Parametern → Logarithmieren / Transformation
  2. Beispiel: I = (U₀/R)·e^(−t/(RC)) → ln(I) = ln(U₀/R) + (−1/(RC))·t
  3. Jetzt ist ln(I) linear in den neuen Parametern → normales Vorgehen wie oben
  4. Am Ende Rücktransformation (exp) für die gesuchten physikalischen Grössen
Linearisierung verändert die Gewichtung der Fehler — nur bei kleinen Residuen vertretbar!
Cholesky LDLᵀ (Normalgleichungen, von Hand)
  1. Normalgleichung: AᵀA · p = Aᵀb
  2. AᵀA ist symmetrisch positiv definit → Cholesky-Zerlegung möglich
  3. LDLᵀ-Zerlegung: AᵀA = L·D·Lᵀ (L = untere Dreiecksmatrix, D = Diagonalmatrix)
  4. Lösung in 3 Schritten: Vorwärtseinsetzen (Lz=b), Diagonalschritt (Dy=z), Rückwärtseinsetzen (Lᵀp=y)
4 · Nichtlineare Gleichungssysteme (Newton-Verfahren)
Newton
Worum geht es?

Zwei oder mehr nichtlineare Gleichungen in mehreren Unbekannten. Newton linearisiert das System iterativ mit der Jacobi-Matrix und konvergiert quadratisch gegen eine Lösung.

Schlagwörter in der Aufgabe
  • nichtlineares Gleichungssystem
  • reelle Lösungen berechnen
  • ganzzahlige Startwerte
  • tol = 10⁻⁹
  • Anzahl Schritte angeben
  • erste zwei Schritte angeben
  • Hinweis auf Lösungsort (Kreisscheibe, Quadrant)
Gegeben / Gesucht
Gegeben System g₁(x,y)=0, g₂(x,y)=0
Gegeben Startwerte, Toleranz

Gesucht Alle reellen Lösungen
Gesucht Werte nach Schritt 1 und 2
Gesucht Anzahl Schritte bis Konvergenz
Kochrezept — Newton für Systeme
  1. Residuenvektor aufschreiben:
    g(r) = [g₁(x,y), g₂(x,y)]ᵀ
  2. Jacobi-Matrix berechnen (alle partiellen Ableitungen):
    J = | ∂g₁/∂x ∂g₁/∂y | | ∂g₂/∂x ∂g₂/∂y |
  3. Newton-Iteration:
    δ = J(rₙ)⁻¹ · g(rₙ) rₙ₊₁ = rₙ − δ
  4. Abbruch wenn ‖δ‖ < tol
  5. Verschiedene Startwerte für verschiedene Lösungen!
Im Praktikum: newtonSystems(g, dg, r0, tol) aufrufen
Pro Lösung einen anderen Startwert → Hinweis im Aufgabentext beachten!
Für die Prüfung: ersten Schritt von Hand ausrechnen = Schritt 1 und 2 zeigen.
5 · Nichtlineare Ausgleichsrechnung (Gauss-Newton)
Gauss-Newton
Worum geht es?

Wie lineare Ausgleichsrechnung, aber das Modell ist nichtlinear in den Parametern. Gauss-Newton linearisiert iterativ mit der Jacobi-Matrix.

Schlagwörter in der Aufgabe
  • nichtlineares Ausgleichsproblem
  • Gauss-Newton-Verfahren
  • Startwerte A₀, B₀, C₀
  • Jacobi-Matrix angeben
  • überbestimmtes nichtlineares System
  • tol = 10⁻⁹
  • Werte nach Schritt 1 und 2
Gegeben / Gesucht
Gegeben Datenpunkte (xᵢ, yᵢ)
Gegeben Modellfunktion φ(x; A,B,C)
Gegeben Startwerte für Parameter

Gesucht Überbestimmtes System f(p) = 0
Gesucht Jacobi-Matrix von f
Gesucht Parameter A, B, C nach Iteration
Kochrezept
  1. Residuenvektor aufstellen (Grösse: Anzahl Datenpunkte):
    fᵢ(p) = φ(xᵢ; p) − yᵢ für alle i
  2. Jacobi-Matrix J (Grösse: n_Daten × n_Parameter):
    J[i,j] = ∂fᵢ/∂pⱼ = ∂φ(xᵢ;p)/∂pⱼ Jede Spalte = partielle Ableitung nach einem Parameter
  3. Gauss-Newton-Schritt:
    JᵀJ · Δp = −Jᵀf(p) p_neu = p + Δp
  4. Abbruch wenn ‖Δp‖ < tol
JᵀJ ist ein kleines quadratisches LGS (Grösse: Anzahl Parameter) — viel kleiner als das überbestimmte System!
6 · Gewöhnliche DGL — Einschrittverfahren (Euler, RK4)
ODE / AWP
Worum geht es?

Anfangswertproblem (AWP): DGL mit Startwert y(x₀) = y₀. Schrittweise Näherung der Lösung von x₀ bis xₙ. Verschiedene Verfahren haben unterschiedliche Genauigkeit (Ordnung) und Stabilität.

Schlagwörter in der Aufgabe
  • Anfangswertproblem (AWP)
  • y(x₀) = y₀ (Anfangsbedingung)
  • Runge-Kutta / RK4
  • explizites / implizites Euler
  • N Schritte, Schrittweite h
  • globaler Fehler an der Endstelle
  • Konvergenzordnung bestimmen
  • Butcher-Tableau
Gegeben / Gesucht
Gegeben DGL y' = f(x, y)
Gegeben Anfangswert y(x₀) = y₀
Gegeben Endpunkt X, Anzahl Schritte N
Gegeben Referenzwert y(X) (exakt)

Gesucht Numerische Lösung y_N ≈ y(X)
Gesucht Fehler |y_N − y(X)|
Gesucht Konvergenzordnung p
DGL 2. Ordnung → System 1. Ordnung (immer erster Schritt!)
y'' = g(x, y, y') → u = [y, y'], u' = [y', g(x, y, y')]
Anfangswert: u(x₀) = [y(x₀), y'(x₀)]. Lösung: u[0] = y(x)
Verfahren im Überblick
  1. Explizites Euler (Ordnung 1, einfach, kann instabil sein):
    y_{n+1} = yₙ + h · f(xₙ, yₙ)
  2. Implizites Euler (Ordnung 1, stabil, Newton-Schritt nötig):
    y_{n+1} = yₙ + h · f(x_{n+1}, y_{n+1})
  3. Mittelpunktsregel (Ordnung 2, explizit):
    k₁ = f(xₙ, yₙ) k₂ = f(xₙ+h/2, yₙ+h/2·k₁) y_{n+1} = yₙ + h·k₂
  1. RK4 (Ordnung 4, Standard):
    k₁ = f(xₙ, yₙ) k₂ = f(xₙ+h/2, yₙ+h/2·k₁) k₃ = f(xₙ+h/2, yₙ+h/2·k₂) k₄ = f(xₙ+h, yₙ+h·k₃) y_{n+1} = yₙ + h/6·(k₁+2k₂+2k₃+k₄)
Konvergenzordnung p: Fehler halbiert sich bei h→h/2 um Faktor 2^p. Euler: 2^1=2, RK4: 2^4=16
Butcher-Tableau lesen
c | A | b ────────────── Explizit: A strikt untere Dreiecksmatrix Implizit: A hat Diagonaleinträge ≠ 0 Anzahl Stufen = Anzahl Zeilen von A Koeffizienten: c_i = Zeitpunkt des Schrittes a_ij = Gewichte der Steigungen b_i = finale Gewichtung
  1. Tableau ablesen → cᵢ, aᵢⱼ, bᵢ identifizieren
  2. Steigungsform aufschreiben: kᵢ = f(xₙ + cᵢ·h, yₙ + h·Σⱼ aᵢⱼ·kⱼ)
  3. y_{n+1} = yₙ + h·Σᵢ bᵢ·kᵢ
Diagonale ≠ 0 → implizit (DIRK). Prüfen ob kᵢ von sich selbst abhängt!
7 · Stabilität von ODE-Verfahren
Stabilität
Worum geht es?

Ab wann "explodiert" die numerische Lösung? Bei zu grosser Schrittweite h kann ein Verfahren instabil werden und riesige Fehler erzeugen, obwohl die exakte Lösung beschränkt bleibt.

Schlagwörter in der Aufgabe
  • Stabilitätsbedingung aufstellen
  • Jacobi-Matrix J(t)
  • Eigenwerte λ von J
  • explizites Euler instabil
  • Schrittweite h₁ stabil, h₂ instabil
  • Verhalten der numerischen Lösung
  • Modellgleichung nichtlinearer Schwingkreis
Gegeben / Gesucht
Gegeben DGL-System (oft nichtlinear)
Gegeben Parameter b, Startwert
Gegeben Zwei Schrittweiten h₁, h₂

Gesucht System 1. Ordnung (Umschreiben)
Gesucht Jacobi-Matrix J(t)
Gesucht Stabilitätsbedingung für h
Gesucht Interpretation der Plots
Kochrezept
  1. DGL 2. Ordnung → System 1. Ordnung:
    u = [x, ẋ], dann u' = [ẋ, x''] explizit
  2. Jacobi-Matrix berechnen:
    J = ∂f/∂u (alle partiellen Ableitungen) An einem Punkt auswerten (z.B. t=0, Startwert)
  3. Eigenwerte λ₁, λ₂ von J berechnen
  4. Stabilitätsbedingung explizites Euler:
    |1 + h·λ| ≤ 1 für alle EW λ → h < 2/|Re(λ)| bei rein imaginären oder negativen EW
Implizites Euler ist unbedingt stabil — funktioniert für alle h (aber Fehler wächst bei grossem h)
h₁ stabil → beschränkte Schwingung im Plot.
h₂ instabil → Lösung wächst exponentiell/explodiert!
Implizites Euler pro Schritt: Newton-Iteration lösen, da y_{n+1} auf beiden Seiten auftaucht.
8 · Systeme von DGL (Aufgabe 9)
ODE-System
Worum geht es?

Mehrere gekoppelte DGL 1. Ordnung in Vektorform: y' = A(x)·y. Euler/RK4 funktionieren genauso, aber mit Vektoren statt Skalaren.

Gegeben / Gesucht
Gegeben Systemmatrix A(x), Anfangsvektor y₀
Gegeben Exakte Lösung (zum Vergleich)
Gegeben Schrittweiten h

Gesucht Absoluter Fehler ‖y_h(X) − y(X)‖ an Stellen X
Gesucht Verhalten bei grossem X (Stabilität)
Kochrezept
  1. f(x, y) = A(x) · y (Matrixmultiplikation!)
  2. Euler / RK4 wie bei skalar, aber y ist ein Vektor
  3. Fehler mit Vektornorm: ‖y_num − y_exakt‖ (Euklidisch)
  4. Bei grossem X mit explizitem Euler → Instabilität möglich!
  5. Implizites Euler stabil aber träger bei grossem h
Explizit bis X=30 mit h=0.1: EW der Matrix prüfen → wenn EW sehr negativ, ist h zu gross!
9 · Kubische Spline-Interpolation
Spline
Worum geht es?

Stützpunkte durch stückweise kubische Polynome verbinden, die an den Übergängen glatt sind (C²-stetig). Die Momente (2. Ableitung an den Knoten) sind die Unbekannten.

Schlagwörter in der Aufgabe
  • kubische Splinefunktion
  • natürliche Randbedingungen
  • periodische Randbedingungen
  • Bedingungsgleichungen aufstellen
  • Spline in Abhängigkeit von Parameter t
  • Momente / zweite Ableitung
Gegeben / Gesucht
Gegeben Stützpunkte (xⱼ, fⱼ)
Gegeben Art der Randbedingung
Gegeben Evtl. Parameter t in den Stützwerten

Gesucht Bedingungsgleichungen
Gesucht Splinefunktion (explizit)
Gesucht Änderungen bei periodischen RB
Kochrezept
  1. Schrittweiten hⱼ = xⱼ₊₁ − xⱼ bestimmen (bei Äquidistanz: alle gleich)
  2. Randbedingungen setzen:
    Natürlich: m₀ = 0, mₙ = 0
    Periodisch: m₀ = mₙ, S'(x₀) = S'(xₙ)
  3. Innere Knoten — tridiagonales LGS: hᵢ·mᵢ₋₁ + 2(hᵢ+hᵢ₊₁)·mᵢ + hᵢ₊₁·mᵢ₊₁ = 6·[(fᵢ₊₁−fᵢ)/hᵢ₊₁ − (fᵢ−fᵢ₋₁)/hᵢ] Bei äquidistant (h=const) vereinfacht sich das!
  4. LGS lösen → Momente m₀…mₙ
  5. Splinewert auswerten auf [xⱼ, xⱼ₊₁]:
    S_j(x) = (mⱼ₊₁(x−xⱼ)³ − mⱼ(x−xⱼ₊₁)³) / (6h) + (fⱼ₊₁/h − mⱼ₊₁·h/6)·(x−xⱼ) + (fⱼ/h − mⱼ·h/6)·(xⱼ₊₁−x)
Bei nur 3 Punkten und natürlichen RB: m₀=0, m₂=0 → nur m₁ unbekannt → eine Gleichung!
Periodisch ≠ natürlich: letzte Zeile des LGS ändert sich (Zyklusbedingung statt m_N=0)
Die rechte Seite des LGS = Differenzen der Steigungen zwischen den Teilintervallen (zweite Differenz der fⱼ)
10 · Lineare Gleichungssysteme — LR-Zerlegung
LGS / LR
Worum geht es?

A·x = b direkt lösen durch Zerlegung in untere (L) und obere (R) Dreiecksmatrix. Dann zwei einfache Dreieckssysteme lösen statt das volle System zu invertieren.

Schlagwörter in der Aufgabe
  • LR-Zerlegung / Gauss-Elimination
  • Vorwärts- / Rückwärtseinsetzen
  • Pivotisierung (Spaltenpivot)
  • LGS lösen
  • Tridiagonalmatrix
  • linsolve / fbsub
Gegeben / Gesucht
Gegeben Matrix A (n×n), rechte Seite b

Gesucht Lösung x mit A·x = b
Gesucht L und R so dass A = L·R
Kochrezept
  1. LR-Zerlegung: Gauss-Elimination auf A anwenden
    → L = untere Dreiecksmatrix (1 auf Diag.)
    → R = obere Dreiecksmatrix
  2. Pivotisierung: vor jedem Schritt Zeile mit grösstem |Eintrag| in Spalte nach oben tauschen → stabiler
  3. Vorwärtseinsetzen: L·y = b lösen (von oben nach unten)
    yᵢ = (bᵢ − Σⱼ﹤ᵢ Lᵢⱼ·yⱼ) / Lᵢᵢ
  4. Rückwärtseinsetzen: R·x = y lösen (von unten nach oben)
    xᵢ = (yᵢ − Σⱼ﹥ᵢ Rᵢⱼ·xⱼ) / Rᵢᵢ
Tridiagonalmatrix: nur 3 Diagonalen besetzt → kompaktere LR-Zerlegung möglich (LR_tri)
Mehrere rechte Seiten b? Zerlegung A=LR einmal berechnen, dann fbsub für jedes b wiederholen — spart Zeit!
Ohne Pivotisierung: kann bei kleinen Diagonaleinträgen numerisch instabil werden
11 · Partielle DGL — Wärmeleitung (FTCS & Crank-Nicolson)
PDE / Wärme
Worum geht es?

Die Wärmeleitungsgleichung ∂T/∂t = a·∂²T/∂x² beschreibt wie sich Temperatur in Zeit und Ort verändert. Diskretisierung in Ort (h) und Zeit (τ) → pro Zeitschritt ein LGS oder explizite Formel.

Schlagwörter in der Aufgabe
  • Wärmeleitungsgleichung
  • FTCS / explizites Schema
  • Crank-Nicolson / implizites Schema
  • Stabilitätszahl s = τ·a/h²
  • Neumann-Randbedingung (Fluss)
  • Dirichlet-Randbedingung (Wert)
  • Temperaturleitfähigkeit a
Gegeben / Gesucht
Gegeben Anfangstemperatur T(x, 0)
Gegeben Randbedingungen (links/rechts)
Gegeben a (Leitfähigkeit), h, τ

Gesucht T(x, t) zu späteren Zeiten
Gesucht Stabilitätsbedingung prüfen
Kochrezept — FTCS (explizit)
  1. Stabilitätszahl berechnen:
    s = τ · a / h² → muss ≤ 0.5 sein!
  2. Zeitschritt explizit:
    T_neu[i] = s·T_alt[i+1] + (1−2s)·T_alt[i] + s·T_alt[i-1]
  3. Randwerte separat behandeln (Dirichlet: fest; Neumann: Fluss einbauen)
s > 0.5 → instabil! Lösung explodiert. Kleineres τ oder grösseres h wählen.
Crank-Nicolson (implizit, immer stabil)
  1. Mittelung zwischen altem und neuem Zeitschritt → immer stabil (kein s-Limit)
  2. Pro Zeitschritt: tridiagonales LGS lösen
    α = a/(2h²), β = 1/τ + a/h² rechte Seite: (1/τ − a/h²)·T_alt + α·(T_alt[i+1]+T_alt[i-1])
  3. Lösen mit scipy.linalg.solve_banded oder np.linalg.solve
Ordnung 2 in Zeit und Ort — genauer als FTCS (Ordnung 1 in Zeit)
12 · Roboterkinematik (Vorwärts- & Rückwärtskinematik)
Kinematik
Worum geht es?

2-Gelenk-Roboter: Vorwärtskinematik berechnet aus Winkeln (θ₁, θ₂) die kartesische Position. Rückwärtskinematik macht das Umgekehrte — nichtlineares Problem, gelöst mit Newton.

Schlagwörter in der Aufgabe
  • Vorwärtskinematik / forwardKin
  • Rückwärtskinematik / backwardKin
  • Gelenkwinkel θ₁, θ₂
  • Greiferposition (x, y)
  • Jacobi-Matrix des Roboters
  • Newton-Iteration auf Gelenken
Gegeben / Gesucht
Vorwärts: Gegeben Winkel θ₁, θ₂
Vorwärts: Gesucht Position (x, y)

Rückwärts: Gegeben Zielposition (x, y)
Rückwärts: Gesucht Winkel θ₁, θ₂
Kochrezept
Vorwärtskinematik (direkt berechenbar)
  1. Ellbogen: P = L₁·[cos θ₁, sin θ₁]
  2. Greifer: T = P + L₂·[cos(θ₁+θ₂), sin(θ₁+θ₂)]
x = L₁·cos(θ₁) + L₂·cos(θ₁+θ₂) y = L₁·sin(θ₁) + L₂·sin(θ₁+θ₂)
Rückwärtskinematik (Newton nötig)
  1. Gleichungssystem aufstellen: g(θ) = forwardKin(θ) − Ziel = 0
  2. Jacobi-Matrix J = ∂g/∂θ (analytisch oder numerisch)
  3. Newton-Iteration: θ_neu = θ − J⁻¹·g(θ)
  4. Startwert: aktueller Winkel oder Schätzwert
Mehrere Lösungen möglich (Ellbogen oben/unten) → Startwert entscheidet!
13 · Adaptive Schrittweitensteuerung (RK4)
Adaptive ODE
Worum geht es?

  • Fehlertoleranz tol
  • einfacher vs. doppelter Schritt
  • Schrittweitensteuerung
  • RK4_Schrittweitensteuerung
  • lokaler Fehler schätzen
Gegeben / Gesucht
Gegeben DGL, Anfangswert, Toleranz tol

Gesucht Lösung mit automatisch variierendem h

Kochrezept — Schrittweitensteuerung (Schritt-Verdopplungs-Methode)
  1. Einen Schritt der Grösse h: y₁ = RK4(xₙ, yₙ, h)
  2. Zwei Schritte der Grösse h/2: y₂ = RK4(RK4(xₙ, yₙ, h/2), h/2)
  3. Fehler schätzen:
    err = ‖y₂ − y₁‖ / 15 (Richardson-Extrapolation für RK4)
  4. Schritt akzeptieren wenn err ≤ tol → xₙ₊₁ = xₙ + h, yₙ₊₁ = y₂
  5. Neue Schrittweite:
    h_neu = h · (tol / err)^(1/5) · Sicherheitsfaktor (≈ 0.9)
  6. Falls err > tol: Schritt verwerfen, h verkleinern, wiederholen
Im Praktikum: RK4_Schrittweitensteuerung_Systems(f, x0, y0, xn, tol) aufrufen
14 · Nichtlineare Ausgleichsrechnung — Levenberg-Marquardt
LM
Worum geht es?

Robustere Alternative zu Gauss-Newton. Kombiniert Gauss-Newton (schnell nahe der Lösung) mit Gradientenabstieg (stabil weit weg). Ein Dämpfungsparameter μ steuert das Verhalten.

Schlagwörter in der Aufgabe
  • Levenberg-Marquardt / LM
  • Dämpfungsparameter μ
  • Gütekoeffizient ρ
  • β₀, β₁ (Schwellen für μ)
  • robuster als Gauss-Newton
  • schlechte Startwerte
Gegeben / Gesucht
Gegeben Daten, Modellfunktion, Startwerte
Gegeben μ, β₀, β₁ (oder Defaults)

Gesucht Optimale Parameter
Gesucht Residuum ‖f(p)‖²
Kochrezept — Unterschied zu Gauss-Newton
  1. Gauss-Newton-Schritt:
    JᵀJ · Δp = −Jᵀf
  2. LM-Schritt (mit Dämpfung μ):
    (JᵀJ + μ·I) · Δp = −Jᵀf
  3. p_neu = p + Δp, Güte ρ berechnen
  4. ρ < β₀ → μ erhöhen (mehr Gradientenabstieg)
  5. ρ > β₁ → μ verringern (mehr Gauss-Newton)
μ → 0: verhält sich wie Gauss-Newton (schnell)
μ → ∞: verhält sich wie Gradientenabstieg (stabil)
Wann LM statt Gauss-Newton? Wenn Gauss-Newton nicht konvergiert oder schlechte Startwerte vorliegen
Im Praktikum: LM(f, J, p, beta0, beta1, mu, maxit)

Kochrezept
  1. Iteration:
    x_{n+1} = xₙ − f(xₙ) / f'(xₙ)
